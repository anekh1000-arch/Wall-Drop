Owner — add wallpapers

No upload page. Copy images into a folder, run one update, refresh the site. Owner-only workflow.

  1. Put images in the right folder
    Desktop
    images\wallpapers\desktop\
    Landscape · PC
    Mobile
    images\wallpapers\mobile\
    Portrait · phones
  2. Update the gallery list
    Double-click update-gallery.bat in the Webpage folder
    or in PowerShell: 
    cd D:\Webpage
python sync.py
    (use py sync.py if python does not work)
  3. View the site
    cd D:\Webpage
python -m http.server 8080
    Open localhost:8080

Optional: set category from the filename

Prefix the file name so filters work without editing JSON:

dark--monterey-black.jpg
minimal--slate-lines.jpg
abstract--void.jpg

Allowed prefixes: dark minimal abstract monochrome gradient

No prefix → treated as dark. Title is taken from the filename.

Any image size is OK — the site reads real width×height automatically (no need to resize to 4K).

Third-party tools (if you want easier management later)

  • Decap CMS (free) — edit a static site in the browser; pairs with Netlify or GitHub. Good if you host on Netlify.
  • Cloudinary (free tier) — store images in the cloud; you paste URLs into JSON. Handy for many large files.
  • Google Drive / Dropbox — only for backup; the site still needs files in desktop / mobile or URLs in JSON.

For now, folder + update-gallery.bat is the simplest and works forever on your PC.
